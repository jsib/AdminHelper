I. AdminHelper (веб-приложение) - вывод информации о работе служб RCO в ИАС и Buzz
	Описание
		а) вывод информации о состоянии консъюмеров (активен/нет) в RabbitMQ для ИАС и Buzz
		б) вывод информации о состоянии служб RCO для ИАС
		  (доступны статусы StartPending, Stopped, Running, ProcessDied, ProcessIsFinishStarting, ProcessIsHanged_KillIt, ServiceIsStopped_StartIt)
	
	Конфигурация
		а) Конфигурация располагается в файле Web.Config
		б) БД для хранения информации
		
	Расположение
		а) Сервер SMI-V-URL01
		б) Сайт AdminHelper.mlg.ru в IIS
		в) Папка %mlg_root%\WebSites\AdminHelper.mlg.ru

II. ManageRcoKernel.ps1 - отслеживает состояние сервисов RCO в ИАС
	Описание
		а) записывает собранную информацию в лог и БД
		б) запускает отстановившиеся сервисы
		в) выполняет поиск зависших процессов, которые перестали набирать кол-во оперативной памяти в процессе загрузки и удаляет процесс
		
	Конфигурация
		а) Конфигурация располагается в самом скрипте
		б) ServerList - список серверов RCO
		в) LogPath - файл лога
		г) MssqlInstance - сервер и порт БД для хранения информации по сервисам
		д) MssqlDatabase - имя БД
		е) timeout_min - таймаут в минутах, через сколько скрипт выполнит следующую итерацию
		
	Тип запуска
		а) Задание в планировщике в папке \MLG 
		б) Задание запущено постоянно
		в) Для бесперебойной работы на случай отказа скрипта выстевлен тригер ежедневного запуска в 15:00, если скрипт не запущен
		
	Расположение
		а) Сервер SMI-V-NERQ01
		б) Папка %mlg_root%\!Scripts\ManageRCOKernels\
		
III. RCOFoldersUpdateStatus.ps1 - проверка обновления папки с объектами RCO
	 Описание
		а) сбор информации по перечню серверов RCO
		б) сравнение размера папки-источника с целевыми папками на серверах
		в) запись информации в лог
		г) оповещение (письмо на servicedesk) если критичное количество серверов не были правильно обновлены
		
	Конфигурация
		а) ServerList - список серверов RCO
		б) LogPath - пусть к файлу лога
		в) NerHLDataPath - путь к данным, генерируемым утилитой Mlg.NerHighlighter.exe buildsnapshot
		г) From, To, SmtpServer, BodyAsHtml, Encoding - настройки почтового клиента

	Тип запуска
		а) Задание в планировщике в папке \MLG 
		б) Задание запускается раз в сутки
		
	Расположение
		а) Сервер SMI-V-NERQ01
		б) Папка %mlg_root%\!Scripts\RCOFoldersUpdateStatus

IV. RCOKernelUpgrade.ps1 - обновление сервисов RCO в ИАС:
	Описание
		а) обновление происходит из заданной на сервере папки
		б) скрипт выполняет предварительную остановку служб и последующий запуск
		в) является заменой ранее использовавшемуся механизму Shadowing
		  (больший контроль над тем, какие сервисы остановлены/запущены в данный момент, на каких серверах обновление уже выполнено по сравнению с Shadowing)
		г) обновление в три этапа по спискам
	
	Конфигурация
		а) ServiceName - имя сервиса RCO
		б) ProcessName - имя процесса
		в) SourceDir - папка-источник
		г) TargetDirLocal, TargetDir - целевая директория
		д) LogPath - путь к файлу лога
		
	Тип запуска
		а) Три задания в планировщике, соответствующие спискам серверов в папке \RCO-Upgrade
		б) Запуск вручную
		
	Расположение
		а) Сервер SMI-V-NERQ01
		б) Папка %mlg_root%\!Scripts\RCOKernelUpgrade

V. 	Приложения HealthyRco, HealthyRcoBzz - сбор информации о состоянии консъюмеров RcoIas и RcoBzz
	Описание
		а) сбор информации
		б) запись собранной информации в базу данных
	
	Конфигурация
		а) БД для записи собранной информации
	
	Тип запуска
		а) Задание в планировщике
		б) Запуск раз в сутки с повторением каждые n-минут

	Расположение
		а) Сервер SMI-V-NERQ01
		б) Папка %mlg_root%\Utils\Mlg.HealthyRco
		в) Задание на сервере MSSQL SMI-V-SUPPLY01 "Admin. Clean Healthy Rco tables"
